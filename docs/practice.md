📚 Практика №1: Страница-энциклопедия (Основы HTML)
Проходится после урока 3.3

Твоя задача — создать структуру страницы без дизайна, используя только HTML-теги. Выбери интересную тебе тему: это может быть обзор на любимую игру, фильм или карточка персонажа из фантастической или RealRPG книги (например, опиши инвентарь или навыки героя).

Что нужно сделать:

Создай файл wiki.html и разверни базовую структуру документа (! + Tab).

Добавь главный заголовок <h1> с названием статьи.

Вставь тематическое изображение с помощью тега <img> (не забудь про атрибут alt).

Напиши 2-3 абзаца текста <p>. Обязательно выдели важные термины жирным (<strong>) и курсивом (<em>).

Создай маркированный список <ul> с интересными фактами или характеристиками.

Сделай таблицу <table> (минимум 3 строки и 2 столбца), чтобы красиво оформить данные (например, "Предмет" — "Урон/Защита").

В самом низу добавь ссылку <a> на источник информации, которая будет открываться в новой вкладке.

🖼️ Как должен выглядеть результат:
Белый фон, черный текст. Сверху крупный заголовок, под ним картинка. Текст разбит на абзацы, есть аккуратный список с точками. Таблица имеет базовые границы (если добавил border="1"), а внизу синяя подчеркнутая ссылка.

📝 Практика №2: Анкета новобранца (Формы)
Проходится после урока 4.3

Любому сайту нужно собирать информацию от пользователей. Давай создадим форму регистрации для вступления в гильдию или закрытый клуб!

Что нужно сделать:

Создай файл register.html и добавь заголовок "Заявка на вступление".

Открой тег <form>. В качестве адреса отправки (action) укажи mailto:babamurad2010@yandex.ru, чтобы заявка улетала почтовому клиенту.

Добавь обязательное текстовое поле type="text" для ввода Никнейма.

Добавь поле type="email" для почты. Снабди оба поля правильными <label> и подсказками placeholder.

Сделай выбор класса персонажа (или роли) с помощью радиокнопок type="radio" (Воин, Маг, Инженер и т.д.).

Добавь выпадающий список <select> для выбора сервера или города.

В самом низу поставь чекбокс "Согласен с правилами гильдии" (сделай его обязательным через required).

Добавь кнопку <button type="submit">Отправить заявку</button>.

🖼️ Как должен выглядеть результат:
Страница с формой, где элементы идут друг за другом. Возле каждого поля ввода есть понятная текстовая подпись. Присутствуют круглые переключатели (выбрать можно только один!) и раскрывающийся список. Внизу выделяется кнопка отправки.

🎨 Практика №3: Одеваем каркас (Базовый CSS и Блочная модель)
Проходится после урока 7.3

Пришло время сделать из скучной черно-белой страницы настоящий шедевр. Мы возвращаемся к проекту из первой практики!

Что нужно сделать:

Создай файл style.css и подключи его к твоему файлу wiki.html внутри тега <head>.

Задай для всего <body> приятный цвет фона и подключи красивый шрифт из Google Fonts.

Ограничь ширину основного текста (например, max-width: 800px) и поставь его по центру экрана с помощью margin: 0 auto.

Сделай рамку для твоей главной картинки, добавь ей скругление углов (border-radius) и отступ снизу, чтобы текст на неё не налезал.

Раскрась таблицу: сделай шапку таблицы (<th>) темной с белым текстом, а строкам задай светлый фон.

Настрой внутренние отступы (padding) для абзацев и списков, чтобы тексту не было "тесно".

🖼️ Как должен выглядеть результат:
Страница больше не прилипает к краям монитора — она аккуратно отцентрирована. Текст читается легко благодаря современному шрифту и увеличенному межстрочному интервалу. Картинка имеет мягкие углы, а таблица выглядит как в профессиональных панелях управления.

🧩 Практика №4: Витрина товаров (Flexbox)
Проходится после урока 9.3

Современные сайты строятся на сетках. Твоя задача — создать адаптивный ряд из карточек, которые выстраиваются идеально ровно благодаря магии Flexbox.

Что нужно сделать:

Создай shop.html и файл стилей к нему.

Сделай общий <div> с классом .container. Задай ему display: flex и выравнивание элементов так, чтобы между ними было одинаковое расстояние.

Внутри контейнера создай 3 одинаковых <div> с классом .card.

Внутри каждой карточки должно быть: картинка предмета, название <h3>, цена и кнопка "Купить".

Настрой стили карточки (.card): задай ей ширину (например, 30%), белый фон, внутренние отступы (padding) и легкую серую рамку.

Сделай саму карточку тоже flex-контейнером, но с направлением flex-direction: column, чтобы ее содержимое выстроилось ровно сверху вниз. Выровняй текст внутри по центру.

🖼️ Как должен выглядеть результат:
Горизонтальный ряд из трех карточек товаров. Карточки имеют одинаковую ширину и стоят на одной линии. Внутри каждой карточки элементы выровнены ровно по центру. Если изменить размер окна браузера, расстояние между карточками пропорционально меняется.

✨ Практика №5: Оживление интерфейса (Анимации)
Проходится после урока 10.2

Сайт должен откликаться на действия пользователя. Добавим немного динамики к нашей витрине товаров из предыдущего задания!

Что нужно сделать:

Открой проект с карточками из Практики №4.

В классе .card добавь свойство transition, чтобы все изменения происходили плавно (например, за 0.3s).

Создай правило .card:hover. При наведении карточка должна:

Слегка приподниматься вверх (используй transform: translateY()).

Получать красивую объемную тень (box-shadow).

Настрой плавную анимацию для кнопки "Купить": при наведении на неё (:hover) она должна менять цвет фона и немного увеличиваться в размерах (transform: scale()).

🖼️ Как должен выглядеть результат (Коллаж из двух состояний):
Слева: Обычная карточка с плоским дизайном.
Справа (имитация курсора): Та же карточка, но она визуально "всплыла" над страницей, под ней появилась реалистичная тень, а кнопка стала ярче и крупнее, приглашая к клику. Изменения между этими состояниями происходят не резко, а плавно.
