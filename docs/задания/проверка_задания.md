3. Перенести проверки прямо в Laravel (Без внешнего сервера)
Если у вас обычный виртуальный хостинг и вы не можете запустить отдельный Node.js сервер, проверку можно переписать на PHP! Вместо отправки HTTP-запроса через RunnerClient, ваш SubmissionScoringService может:

Взять HTML-код студента.
Загрузить его в PHP-библиотеку Symfony\Component\DomCrawler\Crawler (она уже встроена в Laravel).
Прямо в PHP проверить наличие нужных классов: $crawler->filter('.card')->count() > 0.
Что выбрать? Если вы планируете запускать сложные JavaScript проверки студентов — лучше сделать Node.js микросервис. Если задания будут простыми (сверстать карточку, добавить нужный CSS-класс, написать правильный текст в теге <h1>) — я могу помочь вам переписать RunnerClient, чтобы он проверял код прямо внутри Laravel с помощью PHP DomCrawler. Это избавит вас от необходимости искать и настраивать сторонние сервера.


Как теперь писать тесты (Test Cases) в админке:
Когда вы создаете или редактируете "Практику", в блоке добавления тестов нужно заполнять поле script в формате JSON.

1. Проверка HTML (Тип dom) Вы можете проверить, есть ли элемент на странице и содержит ли он нужный текст.

Есть ли карточка? {"selector": ".card", "exists": true}
Есть ли заголовок H2 внутри карточки с определенным текстом? {"selector": "h2", "exists": true, "text": "Заголовок карточки"}
Проверка, что студент не использовал тег <b> (запрещенный тег): {"selector": "b", "exists": false}
(Поддерживаются селекторы: tag, .class, #id, tag.class)

2. Проверка CSS (Тип css) Проверяет наличие свойств в написанном CSS-коде.

Проверка цвета фона у карточки: {"selector": ".card", "property": "background-color", "value": "red"}
Проверка отступов: {"selector": ".card", "property": "padding", "value": "20px"}
Как это протестировать:
Создайте/отредактируйте практику в админке, добавьте тест-кейс (например, dom -> {"selector": ".card", "exists": true}).
На сервере обязательно сбросьте кэш очереди, чтобы Laravel подхватил мой новый код RunnerClient: php artisan queue:restart
Попробуйте сдать задание студентом. Сначала отправьте пустой код (должна появиться ошибка "Ожидался элемент .card"), а затем код с <div class="card"></div> (задание должно пройти проверку).
Всё работает полностью локально и безопасно! Если в будущем понадобятся более сложные проверки (например, клики по кнопкам в JS), то этот механизм всегда можно будет переключить обратно на внешний сервер, просто выключив мок-режим в .env.
